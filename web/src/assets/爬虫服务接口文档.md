# 爬虫服务接口文档（前端接入版）

本文档基于当前代码实现整理，面向前端同学，重点说明：
- 接口怎么调
- 页面如何映射接口
- 任务执行、页面日志、页面库如何联动

## 1. 基本信息

- 容器内服务地址：`http://crawler:5060`
- 本机访问地址（docker-compose 映射）：`http://8.148.22.98:18060`
- 统一前缀：`/crawler-api`
- 数据格式：`application/json`

示例：`http://8.148.22.98:18060/crawler-api/health`

## 2. 认证与跨域

- 当前服务未启用鉴权（无 token 校验）
- CORS 允许全部来源（`*`）

## 3. 状态约定

### 3.1 作业状态（job）
- `pending`：排队中
- `running`：执行中
- `success`：执行成功
- `failed`：执行失败

### 3.2 页面状态（job page）
- `running`：页面抓取中
- `success`：页面抓取成功
- `failed`：页面抓取失败
- `skipped`：已去重，本次跳过

### 3.3 触发来源（trigger_source）
- `manual`：手动触发任务
- `schedule`：定时触发
- `adhoc`：单次抽取接口触发（`/extract`）
- `unknown`：历史数据或缺失字段

## 4. 核心接口总览

### 4.1 健康检查
- `GET /crawler-api/health`

### 4.2 单次抽取（异步）
- `POST /crawler-api/extract`
- `GET /crawler-api/extract/{job_id}`

### 4.3 监控任务管理
- `GET /crawler-api/tasks`
- `POST /crawler-api/tasks`
- `PUT /crawler-api/tasks/{task_id}`
- `POST /crawler-api/tasks/{task_id}/run`
- `POST /crawler-api/tasks/{task_id}/toggle`
- `PUT /crawler-api/tasks/{task_id}/status`
- `DELETE /crawler-api/tasks/{task_id}`
- `GET /crawler-api/tasks/{task_id}/results`

### 4.4 执行任务（Job）
- `GET /crawler-api/jobs`
- `GET /crawler-api/jobs/{job_id}/pages`

### 4.5 日志
- `GET /crawler-api/logs`
- `GET /crawler-api/logs/export`

### 4.6 页面库
- `GET /crawler-api/results`

### 4.7 统计概览
- `GET /crawler-api/dashboard/stats`
- `GET /crawler-api/dashboard/chart?period=7d|30d`
- `GET /crawler-api/dashboard/alerts`

## 5.2 `POST /crawler-api/extract`

用途：临时单次抽取（不依赖任务配置）。

请求体：

```json
{
  "url": "https://www.nea.gov.cn/hudong/gongshi/index.htm",
  "json_schema": "{\"title\":\"string\",\"content\":\"markdown\"}",
  "options": {
    "html_to_markdown": true,
    "remove_scripts_styles": true,
    "include_images": true,
    "cache_mode": "bypass",
    "wait_until": "domcontentloaded",
    "page_timeout": 60000,
    "wait_for": null,
    "only_text": false,
    "remove_forms": false,
    "exclude_external_links": false,
    "simulate_user": false,
    "magic": false,
    "user_agent": null
  }
}
```

响应：

```json
{
  "job_id": "8f4d8f5e6a6f4b74a8d5b2f84e9a0d66",
  "status": "pending",
  "created_at": "2026-03-03T12:00:00.000000"
}
```

查询结果：`GET /crawler-api/extract/{job_id}`

## 5.3 `GET /crawler-api/tasks`

查询参数：
- `q`：任务名/URL 模糊搜索（可选）
- `status`：`active` / `paused`（可选）
- `page`：默认 1
- `size`：默认 10，最大 100

返回字段重点：
- `mode`：`auto | scrape | crawl | list`
- `detail_url_pattern`：详情链接过滤正则
- `options`：抓取配置（与 `ExtractOptions` 一致）
- `last_run_time` / `last_items_count`

---

## 5.4 `POST /crawler-api/tasks`

建议前端统一用新字段：
- `name`
- `url`
- `frequency`（`manual/hourly/daily/weekly` 或 cron）
- `mode`（推荐 `auto`）
- `detail_url_pattern`
- `schema`（对象）
- `options`
- `status`（`active`/`paused`）

示例：

  "name": "能源局政策公告",
  "url": "https://www.nea.gov.cn/policy/gg.htm",

  "schema": {
    "title": "string",
    "publish_date": "string",
    "content": "markdown",
    "source": "string"
  },
  "options": {
    "html_to_markdown": true,
    "remove_scripts_styles": true,
    "include_images": true,
    "cache_mode": "bypass",
    "wait_until": "networkidle",

## 5.8 `GET /crawler-api/jobs`

查询参数：
- `page` / `size`
- `status`（可选）
- `task_id`（可选）

返回重点字段（前端进度条直接可用）：
- `list_page_count`：列表页数
- `discovered_links`：发现链接数
- `effective_links`：过滤后有效链接数
- `total_pages`
- `pending_pages`
- `running_pages`
- `success_pages`
- `failed_pages`
- `skipped_pages`

      "total_pages": 4,

## 5.9 `GET /crawler-api/jobs/{job_id}/pages`

用途：查看某次 job 的页面级执行明细（用于实时日志弹窗）。

查询参数：
- `page` / `size`
- `status`（可选）

返回重点字段：
- `page_url`
- `status`
- `message`
- `started_at` / `finished_at`
- `token_usage`
- `content_markdown`（有正文时可直接渲染）

---

## 5.10 `GET /crawler-api/logs`

查询参数：
- `page` / `size`
- `status`（可选）
- `start_date` / `end_date`（YYYY-MM-DD，可选）
- `job_id`（可选，执行日志联动常用）

返回重点字段：
- `detail_log`：执行详情文本（模式、链接统计、每个页面执行日志）

---

## 5.11 `GET /crawler-api/logs/export`

支持与 `/logs` 相同过滤参数，返回 CSV。

---

## 5.12 `GET /crawler-api/results`

用途：页面库（全量成功爬取页面）。

查询参数：
- `page` / `size`
- `task_id`（可选）
- `q`：标题/URL 模糊搜索（可选）

返回重点字段：
- `task_name`
- `source_url`
- `title`
- `publish_date`
- `data`（含 `content` markdown）

## 6. 前端接入推荐流程

### 6.1 页面初始化
1. 调 `GET /tasks`，渲染监控目标列表。
2. 默认选中一个任务后，调 `GET /jobs?task_id=xx` 渲染作业列表。
3. 调 `GET /logs?page=1&size=50` 渲染日志表。
4. 调 `GET /results?page=1&size=100` 渲染页面库。

### 6.2 手动执行任务
1. 点“立即运行” -> `POST /tasks/{id}/run`。
2. 拿到 `job_id` 后提示用户。
3. 轮询 `GET /jobs?task_id=xx`（2~3 秒）直到该 job 状态结束。
4. 若打开页面日志弹窗，同时轮询 `GET /jobs/{job_id}/pages`。

### 6.3 执行日志联动
- job 行点击“执行日志”：切换日志 tab，并带 `job_id` 调 `GET /logs?job_id=...`。
- 展示 `detail_log`，并提供复制。

### 6.4 页面详情展示
- 优先用 `/jobs/{job_id}/pages` 的 `content_markdown`。
- 页面库中用 `/results` 返回的 `data.content` 渲染 Markdown。

## 7. 常见问题与建议

### 7.1 列表页“发现链接 > 0，但有效链接 = 0”
说明 `detail_url_pattern` 过严，建议：
- 先用宽规则验证：`.*\.htm(l)?$`
- 再收紧为目标站点真实详情 URL 规则

### 7.2 动态渲染列表页抓不到链接
建议配置：
- `wait_until: networkidle`
- `wait_for: 列表容器中的 a 选择器`
- `simulate_user: true`
- `magic: true`
- `page_timeout` 适当增大（如 `120000`）

### 7.3 去重行为说明
同任务下已入库页面会标记为 `skipped`（已爬取，本次跳过），不会重复抓取。

## 8. 前端最小示例（伪代码）

```ts
const API = 'http://127.0.0.1:18060/crawler-api'

async function runTask(taskId: number) {
  const runRes = await post(`${API}/tasks/${taskId}/run`, {})
  const jobId = runRes.job_id

  // 轮询 job 状态
  const timer = setInterval(async () => {
    const jobs = await get(`${API}/jobs`, { page: 1, size: 20, task_id: taskId })
    const cur = jobs.items.find((i: any) => i.job_id === jobId)
    if (!cur) return
    renderJob(cur)
    if (cur.status === 'success' || cur.status === 'failed') {
      clearInterval(timer)
      // 刷日志 + 页面库
      await refreshLogs(jobId)
      await refreshResults(taskId)
    }
  }, 3000)
}
```
